================================================================
  RECALLAI — STEP BY STEP BUILD PLAN
================================================================

A RAG-powered student note app built with LangChain, LangGraph,
FastAPI backend + React Vite with shadcn/ui frontend.

Students organize notes in a simple workspace → files structure.
Each workspace is an isolated context — RAG retrieval is scoped
to the active workspace only.

  Workspace: Biology 101
    ├── lecture_3.pdf
    ├── chapter_summary.pdf
    └── my_notes (text note)

  Workspace: History
    ├── essay_sources.pdf
    └── lecture_notes (text note)

  Upload pipeline (file):
    Browser → POST /api/workspaces/{id}/upload (multipart)
    FastAPI → save to S3 → create DB record (status=processing)
    FastAPI → return {file_id, status} immediately
    BackgroundTask → parse → chunk → embed → OpenSearch
    BackgroundTask → update status=ready in PostgreSQL
    Frontend polls GET /api/files/{id}/status every 2s

  Upload pipeline (text note):
    Browser → POST /api/workspaces/{id}/notes
    FastAPI → save text to S3 as .txt → create DB record
    FastAPI → return {file_id, status} immediately
    BackgroundTask → chunk → embed → OpenSearch
    BackgroundTask → update status=ready in PostgreSQL

  Both pipelines share the same parser/chunker/embedder code.

4 phases, ~8 weeks to a working product.

Step 4 — Build the parsing pipeline (pipeline/)

  parser.py — extract raw text by input type:
    .pdf        → PyMuPDF — text + page number per page
    .pptx       → python-pptx — text per slide
    .docx       → python-docx — paragraphs + tables
    .txt        → read directly
    text_note   → already a string, treat as single page
    Returns: list of {text, page_or_slide}

  chunker.py:
    RecursiveCharacterTextSplitter
    chunk_size=500, chunk_overlap=50
    Preserve page number in each chunk metadata

  embedder.py:
    Embed with all-MiniLM-L6-v2 (384 dimensions)
    Bulk index into OpenSearch "notes" with metadata:
      content, embedding, file_id, file_name,
      workspace_id, workspace_name, page

  run_pipeline() — shared background task:
    async def run_pipeline(file_id, s3_key, file_type, metadata):
        try:
            raw = await s3.download(s3_key)
            pages = parser.parse(raw, file_type)
            chunks = chunker.split(pages)
            await embedder.index(chunks, metadata)
            await db.update_file_status(file_id, "ready")
        except Exception as e:
            await db.update_file_status(file_id, "error", str(e))

Step 5 — Build the file upload route (routes/files.py)

  @router.post("/workspaces/{workspace_id}/upload")
  async def upload_file(
      workspace_id: str,
      file: UploadFile,
      background_tasks: BackgroundTasks,
      db: AsyncSession = Depends(get_db)
  ):
      # 1. Upload to S3
      s3_key = f"workspaces/{workspace_id}/{file.filename}"
      await s3.upload_fileobj(file.file, S3_BUCKET, s3_key)

      # 2. Insert file record (status=processing)
      file_record = await db.create_file(
          name=file.filename,
          s3_key=s3_key,
          workspace_id=workspace_id,
          status="processing"
      )

      # 3. Fire background task — returns to client immediately
      background_tasks.add_task(
          run_pipeline,
          file_id=file_record.id,
          s3_key=s3_key,
          file_type=file.content_type,
          metadata={"workspace_id": workspace_id, ...}
      )

      return {"file_id": file_record.id, "status": "processing"}

  File status endpoint:
    GET /api/files/{id}/status
    → SELECT status, error_reason FROM files WHERE id = ?

  Delete file:
    DELETE /api/files/{id}
    → delete S3 object
    → delete OpenSearch chunks where file_id = ?
    → delete file row from PostgreSQL

  Retry failed file:
    POST /api/files/{id}/retry
    → fetch s3_key from PostgreSQL
    → reset status = "processing"
    → re-trigger run_pipeline() as a new background task

Step 6 — Build the text note route (routes/notes.py)

  @router.post("/workspaces/{workspace_id}/notes")
  async def create_text_note(
      workspace_id: str,
      body: TextNoteInput,    # {title: str, content: str}
      background_tasks: BackgroundTasks,
      db: AsyncSession = Depends(get_db)
  ):
      # 1. Save text as .txt to S3
      note_id = str(uuid4())
      s3_key = f"workspaces/{workspace_id}/notes/{note_id}.txt"
      await s3.put_object(Bucket=S3_BUCKET, Key=s3_key,
                          Body=body.content.encode())

      # 2. Insert file record (file_type=text_note, status=processing)
      file_record = await db.create_file(
          name=body.title,
          s3_key=s3_key,
          workspace_id=workspace_id,
          file_type="text_note",
          status="processing"
      )

      # 3. Same background task as file upload
      background_tasks.add_task(
          run_pipeline,
          file_id=file_record.id,
          s3_key=s3_key,
          file_type="text_note",
          metadata={"workspace_id": workspace_id, ...}
      )

      return {"file_id": file_record.id, "status": "processing"}

Step 7 — Build qa.py with workspace-scoped retrieval

  def retrieve(question: str, workspace_id: str,
               file_id: str = None, k: int = 4):

    scope filter:
      if file_id:   filter on file_id   (single file search)
      else:         filter on workspace_id  (whole workspace)

    OpenSearch KNN query:
      {
        "size": k,
        "query": {
          "bool": {
            "must": {
              "knn": { "embedding": { "vector": [...], "k": k } }
            },
            "filter": [
              { "term": { "workspace_id": "abc123" } }
            ]
          }
        }
      }

  Return: answer + source metadata (file name, page)

================================================================
  HOW THE TWO UPLOAD FLOWS WORK SIDE BY SIDE
================================================================

  FILE UPLOAD:
    Browser → POST /api/workspaces/{id}/upload (multipart)
    FastAPI → boto3 upload → S3 at workspaces/{id}/{filename}
    FastAPI → INSERT files row (status=processing) → PostgreSQL
    FastAPI → background_tasks.add_task(run_pipeline, ...)
    FastAPI → return {file_id, status=processing} instantly
    BackgroundTask → parser → chunker → embedder → OpenSearch
    BackgroundTask → UPDATE status=ready → PostgreSQL
    Frontend polls GET /api/files/{id}/status every 2s

  TEXT NOTE:
    Browser → POST /api/workspaces/{id}/notes {title, content}
    FastAPI → boto3 put_object → S3 at workspaces/{id}/notes/{id}.txt
    FastAPI → INSERT files row (file_type=text_note, status=processing)
    FastAPI → background_tasks.add_task(run_pipeline, ...)
    FastAPI → return {file_id, status=processing} instantly
    BackgroundTask → parser (reads .txt as single page)
    BackgroundTask → chunker → embedder → OpenSearch
    BackgroundTask → UPDATE status=ready → PostgreSQL
    Frontend polls GET /api/files/{id}/status every 2s

  run_pipeline() is identical in both cases.
  file_type tells parser.py which extraction path to use.
